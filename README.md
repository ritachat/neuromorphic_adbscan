## ⚠️ Important Notice — Simulated Data

**All results in this repository are obtained on a synthetic event stream, not real DVS hardware.**

The event stream is generated by `core/event_reader.py` using a physics-based simulator of the DAVIS346 sensor (346×260 px). Object trajectories, event firing patterns, and weather noise (rain/snow) are all procedurally generated. No physical DVS camera or real-world recording was used.

**What this means in practice:**

| Aspect | In this repo | Real DVS hardware |
|--------|-------------|-------------------|
| Event noise | Clean 5% uniform BAC | Hot pixels, refractory noise, shot noise |
| Rain simulation | Directed streaks, fixed width 1–5 px | Variable drop sizes, splash, turbulence |
| Snow simulation | Small diffuse clusters | Partial occlusion, varying flake shapes |
| Object edges | Perfectly sharp bounding box events | Blurred, textured, partially occluded |
| Polarity | Deterministic ON/OFF per edge | Noisy threshold crossings |
| Training data | 62 tracks (GB) / 99 sequences (DL) | Needs 10–100× more for real transfer |

**The 91.2% accuracy result is therefore an upper bound on what would be achievable on real hardware.** Real-world DVS rain and snow produce more varied cluster morphologies, and the polarity flip-rate threshold of 0.28 was calibrated on simulated rain — real rain at different intensities and angles may shift this boundary.

This codebase is intended as a **research prototype and proof-of-concept** to validate the ADBScan algorithm extension and pipeline architecture. Real-data validation is the recommended next step (see [Using Real DVS Data](#using-real-dvs-data) below).

## What This Does

This pipeline detects and classifies moving objects — Car, Truck, Ball, Drone, Pedestrian, Cyclist — from a neuromorphic DVS (Dynamic Vision Sensor) event camera stream under adverse weather: rain (light / medium / heavy), snow (light / medium), and combined Rain+Snow.

**ADBScan** (Adaptive DBSCAN) adapts its search radius ε per event as a quadratic function of radial distance from the sensor centre, correcting for the measured 40% event-density drop from centre to frame edge. A Numba JIT-compiled BFS kernel provides a **4.1× speedup** over sklearn DBSCAN.

Two complementary weather filters are provided:
- **GB filter** — Gradient Boosting on 15 hand-crafted track features (width dominates at 71%)
- **DL filter** — EventResNet, a 984K-param 1D ResNet on raw event sequences (x, y, t, polarity)

A novel **polarity physics filter** (no training) detects rain by polarity flip-rate < 0.28 — a signal invisible to RGB cameras.

## Using Real DVS Data

The detector, filter, and classifier modules are written to accept any `EventStream` object. To run the pipeline on a real DAVIS346 recording:

### 1. Load your events

Replace `EventReader.generate_synthetic()` with a loader for your format:

```python
import numpy as np
from core.event_reader import EventStream

# Example: load from a .npy file with columns [t_us, x, y, polarity]
data = np.load('my_recording.npy')   # shape (N, 4)

stream = EventStream(
    t = data[:, 0].astype(np.int64),   # timestamps in microseconds
    x = data[:, 1].astype(np.int16),   # pixel x  (0 to W-1)
    y = data[:, 2].astype(np.int16),   # pixel y  (0 to H-1)
    p = data[:, 3].astype(np.int8),    # polarity  +1 or -1
    W = 346,                            # sensor width
    H = 260,                            # sensor height
)
labels = np.full(len(stream.t), -1)    # -1 = unknown (no ground truth)
```

### 2. Supported real-data formats

| Dataset | Sensor | Format | Loader |
|---------|--------|--------|--------|
| [DDD20](https://sites.google.com/view/davis-driving-dataset) | DAVIS346 | HDF5 | `h5py` |
| [MVSEC](https://daniilidis-group.github.io/mvsec/) | DAVIS346 | ROS bag | `rosbag` / `tonic` |
| [Slider-SnowDataset](https://github.com/uzh-rpg/rpg_ev-transfer) | DAVIS240C | ROS bag | `tonic` |
| [N-Cars](https://www.prophesee.ai/2018/03/13/dataset-n-cars/) | Metavision | DAT | `metavision_sdk` |
| Live DAVIS346 | DAVIS346 | USB | `dv-python` / `libcaer` |

**Recommended starting point:** [DDD20](https://sites.google.com/view/davis-driving-dataset) — same DAVIS346 sensor as this codebase, recorded in real driving conditions including wet weather.

### 3. Re-train the filters on real data

The GB filter and EventResNet were trained on simulated tracks. For real-data performance, retrain using real labelled recordings:

```python
from core.weather_filter import WeatherClusterFilter
from core.dl_weather_filter import DLWeatherFilter

# Collect real tracks from your labelled scenes
# real_tracks = [...], weather_tracks = [...]

gb = WeatherClusterFilter(mode='gradient_boost')
gb.train(real_track_features, weather_track_features)

dl = DLWeatherFilter(seq_len=256, model_path='results/models/dl_real.pt')
dl.train(real_tracks, real_stream, weather_tracks, weather_stream,
         real_labels, weather_labels)
```

### 4. Expected performance on real data

Based on the simulated results and known sim-to-real gaps in neuromorphic vision:

| Condition | Simulated | Expected real (estimate) |
|-----------|-----------|--------------------------|
| Clean | 100% | 85–95% |
| Heavy Rain | 91% | 70–85% |
| Med Snow | 92% | 65–80% |

The polarity flip-rate filter (threshold 0.28) may need recalibration on real rain data, where raindrop streak widths and durations vary with rainfall intensity and wind direction.

## The Algorithm

### Adaptive ε (US Patent 10,510,154)

```
d(event)  =  √[(x − cx)² + (y − cy)²]
ε(event)  =  clip(c₂·d² + c₁·d + base,  min = 0.9 px)
```

**Optimal coefficients** (grid-searched over 7 simulated weather conditions):

| Coefficient | Value | Meaning |
|-------------|-------|---------|
| c₂ | 0.0002 | curvature of the ε(d) parabola |
| c₁ | −0.030 | slope — vertex position |
| base | 9.5 px | ε at sensor centre |
| Vertex | d = 75 px | minimum ε location |

**Physical justification:** Measured on the synthetic stream, event density drops ~40% from sensor centre (k≈42) to frame edge (k≈25). Adaptive ε compensates for this sparsity.

### Polarity Physics Filter (novel — no training required)

Rain clusters have **flip_rate < 0.28** (measured mean = 0.095 on simulated data):
- A raindrop moves in one direction → sustained same-polarity events → only 1 polarity transition per streak.

Real objects have **flip_rate ≈ 0.47**:
- All 4 bounding box edges fire simultaneously → ON and OFF events intermix.

> **Note:** This threshold was measured on simulated rain. Real-world validation is needed — the threshold may shift depending on rainfall intensity, drop size, and wind angle.

## Key Findings *(on simulated data)*

1. **ADB-Opt 2D + GB = 91.2%** — best of all 28 combinations on simulated data.
2. **GB beats DL for ADBScan; DL beats GB for DBSCAN** — adaptive ε pre-separates clusters spatially, making GB's width feature highly effective.
3. **2D always outperforms 3D** — by 8–23 pp on simulated data.
4. **Full patent Dual (ε+k) = 90.5%** — only 0.7 pp below ε-only Opt.
5. **GB+DL combined = no gain over DL alone.**

## Limitations and Future Work

- **Simulated data only** — all results are on a physics-based synthetic stream. Transfer to real DVS recordings is the critical next step.
- **Small training sets** — GB filter uses 62 tracks, EventResNet uses 99 sequences. Real-data retraining will need significantly more samples.
- **Polarity threshold calibration** — the 0.28 flip-rate threshold for rain detection was tuned on simulated data and needs validation on real rain DVS recordings.
- **Recommended datasets for validation:** DDD20 (same DAVIS346 sensor), MVSEC (driving in rain), Slider-SnowDataset (snow conditions).
